Replacing typed emoticon with user photo

Abstract

In one embodiment, a computing device receives input from a user participating in a message session. The computing device detects an emoticon in the received input and identifies an image corresponding to the emoticon. The computing device accesses the image corresponding to the emoticon and replaces the emoticon with the image in the message session.

Description


TECHNICAL FIELD

This disclosure generally relates to emoticons, in particular, replacing a typed emoticon with a user photo.

BACKGROUND

A social-networking system, which may include a social-networking website, may enable its users (such as persons or organizations) to interact with it and with each other through it. The social-networking system may, with input from a user, create and store in the social-networking system a user profile associated with the user. The user profile may include demographic information, communication-channel information, and information on personal interests of the user. The social-networking system may also, with input from a user, create and store a record of relationships of the user with other users of the social-networking system, as well as provide services (e.g. wall posts, photo-sharing, event organization, messaging, games, or advertisements) to facilitate social interaction between or among users.

The social-networking system may send over one or more networks content or messages related to its services to a mobile or other computing device of a user. A user may also install software applications on a mobile or other computing device of the user for accessing a user profile of the user and other data within the social-networking system. The social-networking system may generate a personalized set of content objects to display to a user, such as a newsfeed of aggregated stories of other users connected to the user.

SUMMARY OF PARTICULAR EMBODIMENTS

In particular embodiments, a computing device, such as a server of a social-networking system, may analyze text of a message entered by a user participating in a message session to detect an emoticon in the text of the message session. The emoticon may, for example, be a character combination such as ":)" representing a smiley face. The computing device, using the detected emoticon, may identify and access an image corresponding to the emoticon. The image corresponding to the emoticon may be an image of the user with an expression similar to the expression conveyed by the detected emoticon. The image might also be any image the user has preselected to represent the expression conveyed by the detected emoticon. The computing device might also prompt the user to select an image. The computing device may provide for display to a second user, such as a recipient of the message, the image in place of the emoticon within the text of the message. The image may, for example, only be displayed to the second user if the second user has permission to view the image.

In particular embodiments, users of social-networking system 160 may communicate with one another via, for example, a messaging service. The messaging service may be hosted by a server 162 of social-networking system 160 and may include a messaging interface that allows users of the social-networking system to send each other messages including text and attachments (e.g. URLs, images, videos, or other files) in a message session. A message session may encompass multiple messages (e.g. multiple lines of text, including attachments) between users. A message session may encompass communication between two users or a plurality of users. A message session may encompass a single communication from a user who posts a message for other interested users of social-networking system 160 to view, such as a status update. In particular embodiments, the messaging interface may also include operability for users of the social-networking system to send each other messages including text and in-line images. An example message in a messaging interface with operability including in-line images is illustrated in FIGS. 3A and 3B. These images may be taken from a variety of sources, including, for example, images available to the users within social-networking system 160, images on third party system 170 available over communication network 110, or images available to client system 130. Images available to client system 130 may also include images from an image capture device, such as a camera, accessible by client system 130.

In particular embodiments, a user of a messaging interface in social-networking system 160 may insert an emoticon into a line of text in a message by specifying a sequence of characters intended to emulate a facial expression. Popular examples are ":)" to represent a smile or ";)" to represent a wink. Emoticons are not limited to facial expressions or a certain number of characters. A more elaborate example is "@>--->---" representing a rose.

In particular embodiments, a computing device such as message server 162 may detect an emoticon in the text of a message by analyzing the text of the received input. Analyzing the text of the received input may include matching a portion of the received text with text of known emoticons stored in data store 164. Data store 164 of known emoticons may be populated by an administrator. In other embodiments, the computing device may populate the known emoticons stored in data store 164 with a user-defined list of emoticons or the computing device may automatically learn emoticons based on prior input of the same or other users. For example, message server 162 may recognize ":)" as an emoticon because the user previously defined that particular string as an emoticon or because the user previously replaced that particular string with an image. Any suitable format for the emoticon may be used to indicate that the text string comprises an emoticon. FIG. 3A illustrates an example in which a single message session includes a user typed emoticon ":)".

Within a line of text in a message, a user may type an emoticon, and a computing device such as message server 162 may replace the emoticon with an in-line image associated with the emoticon. The image may be displayed within the line of text to the user and also to a recipient of the message. The image may be an image of the user with an expression similar to that of the typed emoticon, an image of any person with a similar expression, an image of a pet or other animal with a similar expression, or any image the user or an administrator wants to associate with the emoticon. The image may be a webpage hosted by a third-party system 170 on a server external to social-networking system 160. The image may be an image associated with a concept node 204, a user node 202, or any other object, including, by way of example and without limitation, a profile picture associated with a user, a photo in an album associated with a place or a user, or a specific image chosen by a user (or node administrator) for use as a replacement for an emoticon. The association of emoticons to images may be performed by message server 162 in real time. For example, message server 162 may identify an image by analyzing all images associated with a user node 202 for images with expressions that correspond to the expression of the typed emoticon. Message server 162 may automatically replace the typed emoticon with an identified image, or it may present a selection of images to the user for identification. Message server 162 may store previous user identifications to make automatic future identifications or to present a selection of previously used images to a user for identification. In another example, the user may store a predefined mapping of emoticons to images in data store 164. Message server 162 may identify an image received from an image capture device accessible by client system 130. For example, the user may supply a contemporaneous photo with an expression matching that of the emoticon for message server 162 to identify. FIG. 3B illustrates an example of a single message session in which an image of a man smiling replaces the typed emoticon ":)" from FIG. 3A.

The emoticon typed by the user identifies the image, such that an image associated with the emoticon may be fetched and displayed in-line based on the emoticon. In particular embodiments, an API of the social-networking system may handle the fetching or displaying of images in messages. As an example, a message server 162 of social-networking system 160 may recognize that the sender of a message in a message session has typed an emoticon in the messaging interface. In particular embodiments, message server 162 of social-networking system 160 performs a check to identify an image corresponding to the emoticon. If, for example, no image is associated with the emoticon, message server 162 may display the text string typed by the user instead of displaying an image. In another example, message server 162 may choose to display a default image identified by a system administrator. In particular embodiments, the image displayed in the message may be fixed in size, and in other embodiments, the image may be scaled based upon the text size of the message text.

When the browser or other display interface of the recipient of the message containing an emoticon receives the message, the browser may construct (e.g. in JavaScript) an API query or method call for an image (with arguments based on the emoticon in the message) to a server 162 of social-networking system 160. The server may verify whether the method is valid, whether the recipient user browser has the permission to call the method, and whether the arguments are well-formed and legitimate. Based on this, the server 162 may construct a URL for the image and deliver the URL or image to the browser of the recipient of the message. As described above, the recipient may see a text string or a default image if the emoticon does not have an associated image. Furthermore, the recipient user must have the adequate permissions as stored in social-networking system 160 to see the image. For example, if a sender attempts to send a message with a user profile image that the recipient does not have permission to see, the recipient may see a blank image, or any other suitable image, within the text due to lack of permission.

Furthermore, although particular embodiments involving a messaging service have been discussed, replacing typed emoticons with user photos may be employed throughout social-networking system 160, wherever text is used to communicate. As an example, if a user comments on a post by another user (an example of an edge 206) of social-networking system 160, the comment may include emoticons. Similarly, a post (such as a link or status update) may include emoticons. Additionally, other types of communication services, including email or chat services, may also include operability for replacing typed emoticons with user photos.
